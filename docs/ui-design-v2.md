# AI-Crew UI 설계 v2.0 — 개발 스튜디오

> **목표**: ai-dlc/ooo를 몰라도 "지금 뭘 만들고 있고, 어디까지 왔는지" 바로 파악할 수 있는 팀 공유 UI

작성일: 2026-03-27  
이전 버전: `docs/ui-design.md` (v1.0 — 개인 도구 중심, 영문)

---

## 1. 설계 원칙

### 1.1 핵심 가치

| 원칙 | 설명 |
|------|------|
| **번역 레이어** | ai-dlc/ooo 용어를 일반 개발자 언어로 번역해서 표시 |
| **진행 상태 투명성** | 지금 어떤 단계이고, 무엇이 만들어졌는지 한눈에 |
| **팀 공유 중심** | 혼자만 보는 게 아니라 팀원 누구나 이해 가능 |
| **CLI 실행, UI 확인** | 실행은 CLI, 설계/결과 확인은 UI |
| **한국어 우선** | 모든 UI 텍스트 한국어 기본 제공 |

### 1.2 역할 분리

```
Claude Code CLI          UI (이 문서)
─────────────────        ─────────────────────────
/crew:elaborate    →     설계 단계 진행 상태 확인
/crew:refine       →     설계 문서 열람
/crew:preflight    →     환경 준비 체크리스트
/crew:run          →     개발 진행 칸반 보드
/crew:integrate    →     통합 결과 확인
```

## 2. 정보 아키텍처

### 2.1 내비게이션 구조

좌측 사이드바를 **개발 단계 스텝 네비게이터**로 사용한다.  
ai-dlc 용어 대신 일반 개발자 언어로 매핑한다.

```
📋 설계 단계                  ← Inception Phase
  ├─ 요구사항 분석              ← Requirements Analysis
  ├─ 사용자 시나리오             ← User Stories
  ├─ 아키텍처 설계              ← Application Design
  └─ 작업 분해                 ← Units Generation

🔧 팀 구성                    ← Graph/Bundle 설정
  └─ 에이전트 팀 편집

✅ 개발 준비                  ← Preflight
  └─ 환경 점검

🔨 개발 진행                  ← Run
  └─ 작업 현황 (칸반)

🔗 결과 통합                  ← Integrate
  └─ 통합 현황
```

### 2.2 용어 한글화 대조표

| CLI / 내부 용어 | UI 표시 (한국어) |
|----------------|----------------|
| Inception | 설계 단계 |
| Construction | 개발 단계 |
| Operations | 운영 단계 |
| elaborate | 설계 초안 작성 |
| refine | 설계 고도화 |
| preflight | 개발 환경 준비 |
| run | 개발 실행 |
| integrate | 결과 통합 |
| scratchpad | 작업 메모 |
| bundle | 팀 구성 템플릿 |
| graph | 작업 흐름 |
| node | 에이전트 역할 |
| worker | 작업자 |
| aggregator | 취합자 |
| router | 분배자 |
| hooks | 자동 검증 규칙 |

## 3. 페이지별 상세 설계

### 3.1 설계 단계 뷰 (`/design`)

**목적**: ai-dlc를 모르는 팀원도 "현재 설계가 어디까지 왔고, 무슨 문서가 생성됐는지" 파악

#### 레이아웃

```
┌─────────────────────────────────────────────────────────┐
│ 📋 설계 단계                                             │
│                                                         │
│ [요구사항 분석 ✅] → [사용자 시나리오 ✅] →              │
│ [아키텍처 설계 🔄] → [작업 분해 ⬜]                     │
│                       ↑ 현재 진행 중                    │
├─────────────────────────────────────────────────────────┤
│ 현재 단계: 아키텍처 설계                                  │
│                                                         │
│ 📄 생성된 문서                                          │
│ ├─ requirements.md    2026-03-27    [열기]              │
│ ├─ user-stories.md    2026-03-27    [열기]              │
│ └─ application-design.md (작성 중)                      │
│                                                         │
│ 💡 다음 단계: /crew:elaborate를 실행하면                  │
│    아키텍처 설계가 시작됩니다.                            │
└─────────────────────────────────────────────────────────┘
```

#### 데이터 소스
- `aidlc-docs/aidlc-state.md` 체크박스 파싱 → 단계별 완료 여부
- `aidlc-docs/inception/` 폴더 → 생성된 문서 목록
- 문서 클릭 시 마크다운 렌더링 (모달 또는 우측 패널)

#### 컴포넌트
- **진행 타임라인**: 수평 스텝 UI, 단계별 상태 (완료/진행중/대기)
- **문서 카드**: 파일명, 생성 날짜, 미리보기 첫 줄
- **문서 뷰어**: 마크다운 렌더링 패널 (우측 슬라이드)

---

### 3.2 팀 구성 뷰 (`/team`)

**목적**: 에이전트 팀 구성을 시각적으로 편집

#### 레이아웃

```
┌──────────────────────────────────────────────────────────────┐
│ 🔧 에이전트 팀 구성                                           │
│                                                              │
│  템플릿 선택: [풀스택 팀 ▼]    [템플릿 적용]                  │
├─────────────────────────────┬────────────────────────────────┤
│ 작업 흐름 캔버스             │ 선택된 역할 편집               │
│                             │                               │
│  [기획자]                   │ 역할명: 기획자                 │
│     ↓                       │ 담당자: planner-agent         │
│  [프론트 개발자]──┐          │ 스킬: planning                │
│  [백엔드 개발자]──┤          │ 모델: (기본값)                 │
│                  ↓          │                               │
│             [코드 리뷰어]    │ [저장]  [삭제]                 │
│                  ↓          │                               │
│              [QA]           │                               │
│                             │                               │
│  [+ 역할 추가]              │                               │
└─────────────────────────────┴────────────────────────────────┘
```

#### 인터랙션
- **템플릿 선택** → 팀 구성 자동 생성
- **역할 카드 드래그** → 순서/의존관계 변경
- **팔레트에서 드래그** → 새 역할 추가
- **카드 클릭** → 우측 패널에서 편집
- **저장** → `graph.yaml` 자동 생성

#### 노드 카드 색상 코딩
- 파란색: 작업자 (worker)
- 초록색: 취합자 (aggregator) — 여러 작업 결과를 모음
- 노란색: 분배자 (router) — 조건에 따라 분기

```
┌─────────────────────────────────────────────────────────┐

```

### 3.4 개발 진행 뷰 (`/develop`)

**목적**: 에이전트 실행 현황 및 중간 결과물 확인

#### 탭 구조
- **칸반 보드** (기본) — 에이전트별 작업 상태
- **타임라인** — 시간순 Gantt 차트

#### 칸반 보드 레이아웃

```
┌──────────┬──────────┬──────────┬──────────┐
│ 대기 (2) │ 진행 (1) │ 완료 (2) │ 실패 (0) │
├──────────┼──────────┼──────────┼──────────┤
│          │          │ ╔══════╗ │          │
│ ┌──────┐ │ ┌──────┐ │ ║기획자║ │          │
│ │코드   │ │ │프론트│ │ ║5분  ║ │          │
│ │리뷰어 │ │ │개발자│ │ ║작업  ║ │          │
│ └──────┘ │ └──────┘ │ ║메모  ║ │          │
│          │          │ ╚══════╝ │          │
│ ┌──────┐ │          │ ┌──────┐ │          │
│ │QA    │ │          │ │백엔드 │ │          │
│ └──────┘ │          │ │개발자 │ │          │
│          │          │ └──────┘ │          │
└──────────┴──────────┴──────────┴──────────┘
```

#### 작업 메모(Scratchpad) 뷰어
- 완료된 카드 클릭 → 모달로 마크다운 렌더링
- 에이전트가 작성한 내용 (구현 요약, 의사결정, 이슈) 확인

## 4. 기술 스택

### 4.1 유지할 것

| 항목 | 선택 |
|------|------|
| 프레임워크 | Next.js 15 (App Router) |
| 언어 | TypeScript |
| 데이터 페칭 | SWR |
| 마크다운 렌더링 | react-markdown |
| 그래프 캔버스 | ReactFlow (현행 유지) |

### 4.2 변경할 것

| 항목 | v1.0 | v2.0 |
|------|------|------|
| UI 컴포넌트 | Cloudscape + Tailwind 혼용 | **shadcn/ui + Tailwind** (일관성) |
| 내비게이션 | 플랫 메뉴 | 단계별 스텝 네비게이터 |
| 언어 | 영문 | **한국어 기본** |
| 설계 뷰 | 단순 문서 목록 | 타임라인 + 단계별 맥락 |

> **shadcn/ui 선택 이유**: Cloudscape는 AWS 디자인 시스템으로 enterprise 느낌이 강함.  
> shadcn/ui는 커스터마이징이 자유롭고 한국어 UI에도 자연스럽게 어울림.
