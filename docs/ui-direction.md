# AI-Crew Studio UI — 방향성 문서

> 작성일: 2026-03-29

---

## 목적

CLI(Claude Code)에서 작업하고, UI는 **프로젝트 히스토리를 관찰하는 읽기 전용 도구**.

핵심 질문 3가지에 답하는 것이 UI의 존재 이유:
1. **이전에 어떤 작업을 했는가?**
2. **각 에이전트의 결과물은 어떠했는가?**
3. **그때 설계 문서는 어떤 상태였는가?**

---

## 제약 조건

- **aidlc-docs/는 AI-DLC 소유** — 변형 금지. 읽기만 함.
- **ai-crew 레이어(.ai-crew/)에서만 확장** — 런 격리, 히스토리 관리 등 모든 확장은 여기서.
- **UI는 .ai-crew/를 읽는다** — aidlc-docs는 "현재 최신 문서" 참조 시에만 직접 읽음.

---

## 데이터 설계 방향

### 문제

현재 `.ai-crew/`는 단일 런만 추적. 사이클을 반복하면 이전 결과물이 유실됨.

### 방향

**런별 격리 스토리지**: 각 런이 완료되면 그 시점의 모든 아티팩트(그래프, 노드 상태, 에이전트 결과물, 설계 문서 스냅샷)를 런 디렉토리에 보존.

- `active/` — 현재 진행 중인 런 (실시간 갱신)
- `runs/{runId}/` — 완료된 런 아카이브 (불변)
- 런 아카이브에 aidlc-docs의 **스냅샷**을 포함하여 "그때 설계 문서가 뭐였는지" 재현 가능

### UI가 소비하는 데이터

| 화면 | 필요한 데이터 |
|------|-------------|
| 런 목록 (타임라인) | 런 메타데이터: intent, 상태, 노드 수, 시간 |
| 런 상세 (노드 결과) | 그래프 구조, 노드별 상태, 에이전트 scratchpad |
| 런 시점 설계 문서 | aidlc-docs 스냅샷 (그 런 완료 시점) |
| 현재 설계 문서 | aidlc-docs 직접 (항상 최신) |
| 현재 진행 상태 | active/ (실시간 폴링) |

## UI 설계 방향

### 구조

- **상단 탭 바** (사이드바 아님) — 메뉴가 적으므로 탭이 적절
- **3페이지**: 타임라인(홈), 런 상세, 설계 문서
- **설정은 드로어** — 읽기 전용

### 타임라인 (홈)

- 날짜별 런 카드 목록 (최신 위)
- 진행 중 런은 확장(노드 미니맵), 과거 런은 접힘
- 필터/검색

### 런 상세

- 노드 결과 + 설계 문서를 한 화면에서 탐색
- 노드: 레벨별 그룹핑, 인라인 아코디언으로 scratchpad 열람
- 문서: 그 런 시점의 aidlc-docs 스냅샷 탐색

### 설계 문서 (글로벌)

- aidlc-docs 직접 읽기 (항상 최신 상태)
- 기존 좌측 트리 + 우측 뷰어 구조 유지
