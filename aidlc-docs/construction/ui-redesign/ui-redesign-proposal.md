# UI 전면 리디자인 제안서

> **문서 유형**: 설계 제안서 (Design Proposal)
> **작성일**: 2026-03-29
> **상태**: 검토 대기
> **범위**: AI-Crew Studio UI 전면 재구성

---

## 1. 배경 및 동기

### 1.1 현재 문제

AI-Crew Studio UI는 현재 **7개 페이지**로 구성되어 있으나, 실제 사용 목적과 구조가 불일치한다.

| 페이지 | 실제 사용 빈도 | 문제 |
|--------|---------------|------|
| 대시보드 | 높음 | 스탯 카드 중심 — 히스토리 파악 불가 |
| 설계 단계 | 중간 | 독립적 — 런과 연결되지 않음 |
| 팀 편집 | 낮음 | CLI에서 graph.yaml로 처리 |
| 환경 점검 | 낮음 | CLI에서 /crew:preflight로 처리 |
| 개발 진행 | 중간 | 칸반보드 — 히스토리보다 현재 상태 중심 |
| 팀 템플릿 | 낮음 | CLI에서 번들 전환 |
| 설정 | 낮음 | CLI에서 config.yaml 수정 |

### 1.2 핵심 사용 시나리오

```
사용자가 원하는 것:
  "이전에 어떤 작업을 했고, 현재 어디까지 완료되었으며,
   각 에이전트의 결과물은 어땠는지"를 한눈에 파악

사용자가 하지 않는 것:
  UI에서 에이전트 실행, 설정 변경, 팀 편집
  → 이 모든 작업은 Claude Code CLI에서 수행
```

### 1.3 설계 원칙

1. **UI = 읽기 전용 관찰 도구** — 실행/조작은 CLI의 영역
2. **히스토리 퍼스트** — 과거 → 현재 → 미래 흐름으로 정보 배치
3. **최소 페이지, 최대 깊이** — 페이지 수를 줄이고 드릴다운으로 깊이 확보
4. **문맥적 문서 접근** — 문서는 런과 연결되어야 의미가 있음

## 2. 정보 아키텍처

### 2.1 AS-IS (현재)

```
사이드바 (7개 메뉴)
├── 대시보드          ← 스탯 카드 4개 + 개발 흐름 + 최근 실행
├── 설계 단계         ← 스텝퍼 + 문서 목록 + 마크다운 뷰어
├── 팀 편집           ← 에이전트 팀 구성 (CLI 중복)
├── 환경 점검         ← 프리플라이트 체크 (CLI 중복)
├── 개발 진행         ← 칸반보드 + 실행 기록 탭
├── 팀 템플릿         ← 번들 그리드 (CLI 중복)
└── 설정              ← 프로젝트 설정 (CLI 중복)
```

### 2.2 TO-BE (제안)

```
상단 탭 (2개) + 드릴다운
├── 타임라인 (홈)     ← 프로젝트 히스토리 전체 + 현재 상태
│   └── 런 상세       ← 노드별 결과물 + 생성 문서 (클릭 시 진입)
├── 설계 문서         ← aidlc-docs 전체 탐색
└── ⚙ 설정 (드로어)   ← 우상단 아이콘 → 최소한의 설정 정보 표시
```

### 2.3 제거 대상 및 근거

| 제거 페이지 | 근거 | 대안 |
|-------------|------|------|
| 팀 편집 | graph.yaml은 CLI/에디터에서 수정 | 런 상세에서 에이전트 정보 표시 |
| 환경 점검 | /crew:preflight는 CLI 명령 | 런 상세 또는 타임라인에 결과 표시 |
| 팀 템플릿 | 번들 전환은 CLI 작업 | 프로젝트 현황 바에 현재 템플릿 표시 |
| 설정 | config.yaml은 CLI/에디터에서 수정 | ⚙ 드로어에서 읽기 전용 표시 |

#### 컴포넌트 구성

| 영역 | 컴포넌트 | 데이터 소스 |
|------|---------|------------|
| 상단 바 | `TopBar` | 로고 + 탭 네비게이션 + 설정 아이콘 |
| 프로젝트 현황 | `ProjectSummaryBar` | `/api/config`, `/api/graph`, `/api/aidlc/state` |
| 필터/검색 | `TimelineFilter` | 상태(전체/완료/실패), 텍스트 검색 |
| 날짜 구분선 | `DateDivider` | 런 데이터의 createdAt 기준 그룹핑 |
| 현재 런 카드 | `ActiveRunCard` | `/api/state`, `/api/graph` — 확장 상태 |
| 과거 런 카드 | `CompletedRunCard` | `/api/runs` — 접힌 상태, 한 줄 요약 |
| 실패 런 카드 | `FailedRunCard` | `/api/runs` — 실패 원인 표시 |

#### 런 카드 상태별 표시 규칙

| 상태 | 카드 스타일 | 표시 정보 |
|------|-----------|----------|
| **진행 중** | 확장, 보라색 테두리, 노드 미니맵 | intent, runId, 노드별 상태, 경과 시간 |
| **완료** | 접힘, 회색 배경, 한 줄 | intent, runId, 노드 수, 소요 시간, 문서 수 |
| **실패** | 접힘, 붉은 테두리 | intent, runId, 실패 노드 + 사유 |

#### 노드 미니맵 (현재 진행 중 런에만 표시)

```
각 노드는 작은 사각형 칩으로 표시:
  ✅ 완료 = 초록 배경 + 체크 + 소요시간
  🔵 진행 = 보라 배경 + 펄스 애니메이션
  ⏳ 대기 = 회색 배경
  🔴 실패 = 빨강 배경 + 에러 아이콘
```

#### 레이아웃 구성

**2컬럼 레이아웃 (6:4 비율)**

| 영역 | 위치 | 내용 |
|------|------|------|
| 런 헤더 | 상단 전체 폭 | intent, runId, 시간, 상태, 노드 요약 |

| 문서 패널 | 우측 40% | 생성 문서 목록(상) + 마크다운 미리보기(하) |

#### 노드 결과물 표시 (아코디언)

```
각 노드 카드:
├── 헤더: [상태 아이콘] 노드명  소요시간
│         agent: 에이전트명
├── [결과물 보기 ▾] 클릭 시 펼침:
│   └── scratchpad 내용을 마크다운으로 렌더링
└── 접힌 상태에서도 한 줄 요약 표시
```

#### 문서 패널 동작

1. 상단: 이 런에서 생성/수정된 문서의 트리 목록
2. 하단: 선택된 문서의 마크다운 미리보기
3. 문서 클릭 시 하단 미리보기 영역에 렌더링
4. "설계 문서 탭에서 열기" 링크 제공

| API | 사용 위치 | 용도 |

| `GET /api/aidlc/state` | 설계 문서 | 설계 진행 상태 |

| `GET /api/aidlc/docs?path=` | 설계 문서/런 상세 | 문서 내용 |

| 노드 레벨 정보 | ✅ | graph.yaml의 depends_on으로 계산 |
