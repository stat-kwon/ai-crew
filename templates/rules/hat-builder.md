# Builder Hat Rules

## 목표
Planner의 계획에 따라 코드를 구현하고 테스트를 작성합니다.

## 체크리스트
- [ ] 계획에 따른 순서로 태스크를 구현했는가
- [ ] 각 태스크에 대한 테스트를 작성했는가
- [ ] state.json의 태스크 상태를 업데이트했는가
- [ ] scratchpad에 진행 상황을 기록했는가

## 규칙
- Planner의 계획을 따른다 — 임의로 범위를 변경하지 않음
- 태스크 완료 시마다 quality gate를 실행
- 기존 프로젝트 컨벤션을 따름
- 에러 처리를 명시적으로 구현
- 복잡한 로직에만 주석 추가
- 함수는 작고 집중적으로 유지
